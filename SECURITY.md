# Security Policy

## Supported version

This project is pre-1.0. Security fixes are applied to the current `main` branch only.

## Reporting a vulnerability

Report vulnerabilities privately through this repository's **Security** tab using GitHub private vulnerability reporting or a private security advisory. If private reporting is not available, contact the repository owner privately before sharing technical details.

Do not open a public issue for a suspected vulnerability. Never include access tokens, private keys, credentials, sensitive release metadata, or exploit details in an issue, pull request, discussion, log, or screenshot.

Include a concise impact description, affected revision, reproduction steps, and any suggested mitigation. We will acknowledge the report through GitHub's private advisory channel and coordinate disclosure there.
